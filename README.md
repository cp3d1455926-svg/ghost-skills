## 🚀 Quick Start

### Prerequisites

- [OpenClaw](https://github.com/openclaw/openclaw) installed
- Node.js 18+

### Installation

**Option 1: Clone Repository**

```bash
# Clone repository
git clone https://github.com/cp3d1455926-svg/OpenClaw-good-skills.git

# Create symlink to OpenClaw workspace
# Windows (Admin PowerShell)
New-Item -ItemType SymbolicLink -Path "C:\Users\YourUsername\.openclaw\workspace\skills\ghost-skills" -Target "D:\path\to\ghost-skills"
```

**Option 2: Use ClawHub**

```bash
clawhub install ghost-skills
```

**Option 3: Copy Individual Skills**

```bash
# Copy single skill to workspace/skills
cp -r ghost-skills/official-account-assistant ~/.openclaw/workspace/skills/
```

## 🛠️ Usage

### Skill Structure

Each skill folder has a standard structure:

```
skill-name/
├── SKILL.md           # Required: Skill description and triggers
├── index.js / script.js # Main logic
├── scripts/           # Helper scripts
├── references/        # Reference materials
├── assets/            # Static assets
├── README.md          # Usage documentation
└── _meta.json         # Optional: Metadata
```

### ⚠️ Security Notice

**Always vet skills before installation:**

```bash
# Use skill-vetter to review skills
# Ask in OpenClaw: "Help me vet this skill"
```

**Checklist:**
- Read SKILL.md to understand functionality
- Read README.md for usage instructions
- Check for suspicious shell commands
- Verify permission requests are reasonable
- No hardcoded API keys or credentials

### Creating New Skills

Use `skill-creator` to quickly generate:

```bash
# In OpenClaw
"Help me create a new skill with functionality XXX"
```

### Testing Skills

1. Place skill in `workspace/skills/`
2. Trigger skill in OpenClaw
3. Check logs and output
